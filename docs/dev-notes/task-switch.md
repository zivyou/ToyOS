
# i386进程切换原理

## 时机
i386进行进程切换总结一下就只有两种时机:
1. 当前进程主动调用switch_task(), 相当于当前进程主动放弃CPU资源,切换到其他进程;
2. 在中断处理例程处理结束之后,准备返回到用户态之前; 大家熟悉的通过时钟中断触发本质上也是这个原理,其实任何中断/异常/系统调用都可以触发这个效果;

## 进程上下文(task context)布局
1. i386栈方向为地址逆序: 从高地址往低地址增长;
2. 从低地址往高地址依次记录: entry, EBX, EBP, EDI, ESI, EFLAGS;

## 可抢占与不可抢占
1. 可抢占: 上文👆提到的切换时机2, 如果返回到用户态的时候, 还是返回到其原来的进程,就是不可抢占; 如果返回到用户态的时候,不保证一定回到原来的进程,就是可抢占;
